Image Markup & Automation Workflow Helper

Workflow Overview

Step 1: Manually add your high-resolution master images (e.g., 4K JPEG or PNG) into your project folders such as:
/assets/img/blog/ or /assets/img/works/ (including any subfolders).
Step 2: Run your Node.js Sharp image processing script to generate optimized images:
node generate-thumbnails.js
This script will:
  - Process all JPG, JPEG, and PNG images recursively in blog and works folders.
  - Create resized JPEG and WebP images at set widths (e.g., 600px and 1200px).
  - Save optimized images inside optimized subfolders corresponding to each location.
For example:
/assets/img/blog/optimized/ and /assets/img/works/motion/optimized/.
Step 3: Update your websiteâ€™s HTML image markup to use the optimized images by referencing the optimized folder paths.
See example markup below.

Example HTML Markup for Basic Images (WebP + JPEG fallback)

<picture>
  <source srcset="assets/img/blog/optimized/example-image.webp" type="image/webp" />
  <source srcset="assets/img/blog/optimized/example-image.jpg" type="image/jpeg" />
  <img src="assets/img/blog/optimized/example-image.jpg" alt="Descriptive text here" />
</picture>

Explanation:

  • Browser tries to load WebP first, falls back to JPEG if unsupported.
  • <img> fallback ensures image displays in browsers without <picture> support.
  • Include descriptive alt text for accessibility.

Example Responsive Image Markup with Multiple Sizes

<picture>
  <source
    srcset="assets/img/blog/optimized/example-600.webp 600w, assets/img/blog/optimized/example-1200.webp 1200w"
    sizes="(max-width: 600px) 600px, 1200px"
    type="image/webp"
  />
  <source
    srcset="assets/img/blog/optimized/example-600.jpg 600w, assets/img/blog/optimized/example-1200.jpg 1200w"
    sizes="(max-width: 600px) 600px, 1200px"
    type="image/jpeg"
  />
  <img src="assets/img/blog/optimized/example-1200.jpg" alt="Descriptive text for responsive image" />
</picture>

Explanation:

  • Use srcset with width descriptors to provide multiple resolutions.
  • sizes guides browsers to select appropriately sized images based on viewport.
  • Improves load times and image clarity on different devices.

Additional Tips

  • Keep your master images well organized inside /assets/img/blog and /assets/img/works folders and subfolders.
  • Run the image processing script whenever you add or update high-res images to keep optimized assets fresh.
  • Use consistent naming conventions so optimized images are easily referenced within the optimized folder.
  • Do not serve master images directly on the website; always use the optimized JPEG or WebP versions.
