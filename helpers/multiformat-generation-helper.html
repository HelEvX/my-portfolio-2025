Image Markup & Automation Workflow Helper

Workflow Overview

Step 1: Place your high-resolution master images into your project folders:
/assets/img/blog/ (and subfolders like general, pxl)
/assets/img/works/ (and subfolders)
Step 2: Run the Node.js Sharp script to generate optimized images.

Use the **npm command**: 
npm run images
Options:
  • Process everything: npm run images
  • Process only a specific folder: npm run images -- blog/general
  • Process only a single image: npm run images -- blog/general/introduction-cover.jpg
The script will:
  • Process JPG, JPEG, and PNG images recursively.
  • Skip existing optimized images unless deleted manually.
  • Save results inside assets/img/optimized/, maintaining subfolders:
    • Blog: assets/img/optimized/blog/general/
    • Works: assets/img/optimized/works/graphics/
Step 3: Update your HTML to use the optimized images. For **blog cover images**, only the 600px versions are needed, since your max width is under 550px.

Example HTML Markup for a Blog Cover Image

<picture>
  <source srcset="assets/img/optimized/blog/general/introduction-cover-600.webp" type="image/webp" />
  <source srcset="assets/img/optimized/blog/general/introduction-cover-600.jpg" type="image/jpeg" />
  <img src="assets/img/optimized/blog/general/introduction-cover-600.jpg" alt="Descriptive text for post" />
</picture>
  • Browser tries WebP first, falls back to JPEG if unsupported.
  • Include descriptive alt text for accessibility.
  • No need for 1200px images for blog tiles, only 600px.

Example HTML Markup for Other Images (Works or Full-Width)

<picture>
  <source
    srcset="assets/img/optimized/works/graphics/photo-600.webp 600w, assets/img/optimized/works/graphics/photo-1200.webp 1200w"
    sizes="(max-width: 600px) 600px, 1200px"
    type="image/webp"
  />
  <source
    srcset="assets/img/optimized/works/graphics/photo-600.jpg 600w, assets/img/optimized/works/graphics/photo-1200.jpg 1200w"
    sizes="(max-width: 600px) 600px, 1200px"
    type="image/jpeg"
  />
  <img src="assets/img/optimized/works/graphics/photo-1200.jpg" alt="Descriptive text" />
</picture>

Additional Tips

  • Keep master images organized in /assets/img/blog and /assets/img/works only.
  • Run the script whenever you add or update master images.
  • Delete optimized images if you want to regenerate them.
  • Always reference optimized images in HTML, not the master images.
