HTML Helper for Using WebP Images with JPEG Fallback

Basic Image with WebP + JPEG Fallback

<picture>
  <source srcset="assets/img/example-image.webp" type="image/webp" />
  <source srcset="assets/img/example-image.jpg" type="image/jpeg" />
  <img src="assets/img/example-image.jpg" alt="Descriptive text here" />
</picture>

Explanation:

  • The browser tries to load the first supported format image/webp.
  • If WebP isn’t supported, it falls back to the JPEG source.
  • <img> ensures content is shown on browsers that don’t support <picture>.
  • Always include descriptive alt text for accessibility.

Example with Responsive Images

<picture>
  <source 
    srcset="assets/img/example-600.webp 600w, assets/img/example-1200.webp 1200w" 
    sizes="(max-width: 600px) 600px, 1200px" 
    type="image/webp" />
  <source 
    srcset="assets/img/example-600.jpg 600w, assets/img/example-1200.jpg 1200w" 
    sizes="(max-width: 600px) 600px, 1200px" 
    type="image/jpeg" />
  <img 
    src="assets/img/example-1200.jpg" 
    alt="Descriptive text for responsive image" />
</picture>

Explanation:

  • Use srcset with width descriptors (600w, 1200w) for different resolutions.
  • sizes attribute tells the browser which image size to load based on viewport width.
  • This improves performance and image quality on different screens.

Tips

  • You don’t need to change your CSS targeting the <img> tag inside <picture>.
  • The browser loads only one image—the first supported source it finds.
  • Keep your images organized on disk with matching names for WebP and JPEG versions.
  • Use automated scripts or tools (like Photoshop, ImageMagick, or cwebp) to generate your WebP and JPEG images consistently.
